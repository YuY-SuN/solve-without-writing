# benkyo-tool-prompt01 完了フラグ・ページ進捗プラン

## 目的

問題ごとに「この問題をやった」状態を保存し、その集計によってページごとの進捗を視認できるようにする。

今回のMVPでは、まず以下を実現する。

- 問題ごとの完了フラグを保存できる
- ページごとの完了数と残数を見えるようにする
- 入力漏れがある問題を誤って完了にできないようにする

## 基本方針

- 完了状態は手動で付ける
- ただし、完了を付けられるのは「その問題に必要な入力がすべて埋まっているとき」に限る
- `response` を1つも持たない問題は、最初から「入力済み」とみなす
- 保存先は既存の回答保存と同じく `localStorage` とする

## 用語

### 完了可能

その問題に必要な入力条件を満たしており、手動で完了を付けられる状態。

### 完了

完了可能な問題に対して、ユーザーが手動で「この問題をやった」を付けた状態。

## 問題ごとの状態

各問題は次の3状態で扱う。

1. 未入力または入力不足
2. 完了可能
3. 完了

振る舞いは以下の通り。

- 未入力または入力不足
  - 完了トグルは disabled
  - 補助文で「入力がそろうと完了にできます」を表示する
- 完了可能
  - 完了トグルは enabled
  - まだ進捗の完了数には数えない
- 完了
  - 完了数に加算する

## 入力済み判定

### 原則

問題単位の「必要な入力」は、以下の対象をすべて集めて判定する。

- 問題本体の `response`
- 各 `item` の `response`

既存の `getProblemResponseKeys(problem)` をベースに、各 `responseKey` が入力済みかどうかを判定する。

### `response` が 0 件の問題

- 必要な入力欄はすでに満たしているものとみなす
- 初期状態は「完了可能」
- 完了数に入るのは手動で完了を付けた後

### 型ごとの入力済み条件

- `blank`
  - 空文字ではない
- `multi_blank`
  - すべての `field.key` が空文字ではない
- `choice`
  - 単一選択: 値が空でない
  - 複数選択: 配列が1件以上ある
- `table_fill`
  - `response.targets` に含まれるすべての `key` が空文字ではない
- `draw_graph`
  - `answer.value` が配列なら、その件数と同数の点が入力されている
  - `answer.value` がオブジェクトで `bins` 相当を持つなら、その件数がそろっている
- `draw_point`
  - `answer.value` がオブジェクトなら、そのキー数と同数の点ラベルが入力されている

### 未知または未対応の型

- 将来型が追加されたときに誤って完了できないよう、MVPでは「未入力扱い」に倒す
- UI上も完了不可のままにする

## 保存方式

### 回答

既存の `responseValues` を継続利用する。

### 完了フラグ

`responseValues` とは別キーで保存する。

- 例: `benkyo-tool-prompt01:completed-problems:v1`

保存値は「完了している問題キーの集合」に相当するオブジェクトまたは配列とする。

問題キーは dataset と page を含めて一意化する。

- 例: `math-textbook-pages-12-13::12::p001_04`

## 集計

ページごとに以下を集計する。

- 総問題数
- 完了数
- 残数

必要なら内部的には次も持てる。

- 完了可能だが未完了の数
- 入力不足の数

ただしMVPの見せ方は、まず以下を優先する。

- `完了 x/y`
- `残り z問`

## UI案

### 問題カード

各問題カードの右上に完了トグルを置く。

- ラベル: `完了`
- 入力不足のときは disabled
- disabled のときは補助文を表示

### ページ進捗

ページ単位の進捗は、以下のどちらか、または両方に表示する。

- `ページ` セレクトの選択肢文言
- ツールバー下の進捗一覧

初期MVPでは、進捗一覧を常設する。

表示例:

- `12ページ 3/5 完了`
- `残り 2問`

## 既知の注意点

- 完了トグルは回答内容の正誤を判定しない
- 完了後に回答を消した場合、完了状態は自動解除するのが自然
- 描画系入力は `0` が有効値になりうるため、「truthy かどうか」では判定しない

4. `index.html` と `page.css` に進捗表示UIを追加する
5. 入力削除時の完了解除を含めて動作確認する
