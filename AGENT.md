# AGENT.md

## Documentation Rule

- このリポジトリでは、機能追加・仕様変更・運用ルール変更を行ったら、同じ変更セットの中で `docs/` 配下の関連Markdownも更新すること。
- `docs/` には少なくとも次を残すこと。
  - 何を解決する機能か
  - どう設計したか
  - 実装上の要点と更新対象ファイル
  - 運用手順や更新手順
- UIやデータ仕様に変更が入る場合は、利用者視点の操作説明と、保守者視点のデータ更新手順の両方を書くこと。
- 今回のように独立ツールを追加した場合は、実行場所・入力・出力・副作用を明記すること。
- 継続的に参照する前提知識、技術的制約、作業ルール、Git運用上の注意は `docs/engineering-notes.md` に集約し、変更が出たら更新すること。
