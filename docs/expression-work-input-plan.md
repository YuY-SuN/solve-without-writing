# 式の過程入力プラン

## 何を解決する機能か

問題の最終解答だけでなく、素因数分解、分配法則、通分、約分などの途中過程を、ただの自由記述ではなく式の変形ステップとして入力できるようにする。

この機能は、学習者が「計算を整理して分解する」習慣を身につけるための任意入力欄である。未入力でも従来どおり完了できるが、使う場合は式、関係記号、操作メモを1行ずつ残せる。

## 設計方針

- 最終解答の `response` とは分けて、問題または小問に `work` を追加する。
- 初期実装では汎用の `expression_steps` のみを実装する。
- `expression_steps` は縦方向に式変形を並べる入力UIとする。
- 各ステップは関係記号、式、メモを持つ。
- 途中式は必須ではないため、完了判定には含めない。
- 保存データは `responseValues` と分けて `workValues` に保持する。
- 将来、素因数分解の階段図、通分・約分専用テンプレート、分配法則専用テンプレートへ拡張できるようにする。

## データ仕様

問題本体または小問に次の `work` を追加できる。

```json
{
  "work": {
    "type": "expression_steps",
    "optional": true,
    "label": "考え方・途中式",
    "defaultOpen": true,
    "mode": "fraction",
    "starter": {
      "expression": "1/2 + 1/3"
    },
    "suggestedNotes": ["通分", "分子を計算", "約分"]
  }
}
```

保存値は `workValues` に次の形式で保持する。

```json
{
  "steps": [
    {
      "relation": "",
      "expression": "1/2 + 1/3",
      "note": ""
    },
    {
      "relation": "=",
      "expression": "3/6 + 2/6",
      "note": "通分"
    }
  ]
}
```

## UI仕様

- 問題文・図表の後、最終解答欄の前に途中式欄を表示する。
- 小問の場合は、小問本文・図表の後、解答欄の前に表示する。
- `details` による折りたたみ可能なパネルにする。
- 行ごとに次を入力する。
  - 関係記号: 空、`=`、`≈`、`<`、`>`、`→`、`⇔`
  - 式入力欄
  - メモ欄
- 式入力欄には、分数、累乗、平方根、括弧、乗除、マイナスの補助ボタンを置く。
- メモ候補として、通分、約分、分配法則、移項、同類項をまとめる、計算などを選べる。

## 実装上の要点と更新対象ファイル

- `app/src/renderers/WorkRenderer.js`
  - `expression_steps` 用の入力UIを実装する。
- `app/src/renderers/ProblemRenderer.js`
  - 問題本体と小問の `work` を検出し、解答欄の前に描画する。
- `app/src/main.js`
  - `workValues` の状態、localStorage保存、記憶データexport/import、クリア処理を追加する。
- `app/src/styles/page.css`
  - 途中式欄、式入力、メモ候補、スマホ表示のスタイルを追加する。
- `app/src/data/*.json`
  - 分配法則と通分のサンプル問題に `work` を追加する。
- `README.md`
  - 利用者向け操作とデータ更新手順を追記する。

## 利用者向け操作手順

1. 問題カードに「考え方・途中式」または問題別ラベルの欄が表示されている場合、必要に応じて開く。
2. 最初の行に元の式を書く。
3. `＋ 行を追加` で次の式変形を追加する。
4. 関係記号を選び、式欄に変形後の式を書く。
5. 必要ならメモ欄またはメモ候補で「通分」「分配法則」などを書く。
6. 最終解答欄には従来どおり答えを書く。

途中式欄は任意なので、空欄のままでも完了判定には影響しない。

## 保守者向けデータ更新手順

1. 途中式を入力させたい問題または小問に `work` を追加する。
2. 初期式が明確な場合は `starter.expression` に入れる。
3. 問題の種類に合わせて `mode` と `suggestedNotes` を指定する。
4. 問題JSONを更新したら、必要に応じて `app/src/data/sync_index.py` を実行する。
5. UIまたはデータ仕様を変えた場合は、READMEと関連docsを同じ変更セットで更新する。

## 将来拡張

- `factorization_steps`: 素因数分解の階段図形式の過程入力。
- `fraction_transform_steps`: 通分・約分で共通分母や共通因数を明示する専用入力。
- `distribution_steps`: 分配前、各項への分配、整理後を明示する専用入力。
- 転記モードで「入力内容＋途中式」を印刷できる表示。
- 途中式の簡易検証や、式変形の誤り候補表示。

## ブランチ運用メモ

この実装は `feature/expression-work-input` ブランチで行い、main へはこの作業内でマージしない。
